Display y Visibilidad

Aprende a controlar cómo se muestran los elementos y su visibilidad en CSS

Introducción

Las propiedades de display y visibilidad en CSS controlan cómo se muestran los elementos en la página y cómo interactúan con otros elementos. Estas propiedades son fundamentales para crear layouts y controlar la presentación visual de los elementos HTML.

Propiedad Display

La propiedad display determina cómo se muestra un elemento en el flujo del documento y cómo interactúa con otros elementos.

  • block: Ocupa todo el ancho disponible y crea una nueva línea
  • inline: Ocupa solo el espacio necesario y no crea nuevas líneas
  • inline-block: Combinación de block e inline
  • none: Oculta el elemento y elimina su espacio
  • flex, grid, table: Otros valores para layouts específicos

Propiedades de Visibilidad

Estas propiedades controlan si un elemento es visible o no, sin necesariamente afectar su espacio en el layout.

  • visibility: Controla si un elemento es visible (hidden/visible)
  • opacity: Controla la transparencia de un elemento (0-1)
  • display: none: Oculta el elemento y elimina su espacio

Estas propiedades tienen comportamientos diferentes en cuanto a:

  • Espacio ocupado en el layout
  • Interactividad (eventos de clic, etc.)
  • Accesibilidad y lectores de pantalla

¿Por qué son importantes?

Layouts

Las propiedades de display son fundamentales para crear layouts y estructurar el contenido de una página web.

Interactividad

Controlar la visibilidad de los elementos permite crear interfaces interactivas como menús desplegables, modales y tooltips.

Responsividad

Cambiar el display de los elementos permite crear diseños responsivos que se adaptan a diferentes tamaños de pantalla.

Propiedad Display

La propiedad display determina cómo se muestra un elemento en el flujo del documento y cómo interactúa con otros elementos. Veamos los valores más comunes:

display: block

  • Ocupa todo el ancho disponible de su contenedor
  • Siempre comienza en una nueva línea
  • Respeta propiedades de ancho y alto
  • Respeta márgenes en todos los lados
  • Ejemplos: div, p, h1-h6, section, article, form
.elemento {
    display: block;
    width: 50%;
    height: 100px;
    margin: 10px;
}

display: inline

  • Ocupa solo el espacio necesario para su contenido
  • No crea nuevas líneas
  • Ignora propiedades de ancho y alto
  • Solo respeta márgenes horizontales (left/right)
  • Ejemplos: span, a, strong, em, img
.elemento {
    display: inline;
    /* width y height no tendrán efecto */
    width: 100px;
    height: 100px;

    margin: 10px;

display: inline-block

  • Combinación de block e inline
  • No crea nuevas líneas (como inline)
  • Respeta propiedades de ancho y alto (como block)
  • Respeta márgenes en todos los lados (como block)
  • Ejemplos: button, input, select
.elemento {
    display: inline-block;

display: none

  • Oculta completamente el elemento
  • Elimina el elemento del flujo del documento
  • No ocupa espacio en la página
  • No es accesible para lectores de pantalla
  • Útil para elementos que deben mostrarse/ocultarse dinámicamente
.elemento {
    display: none;
    /* Estas propiedades no tendrán efecto */
    width: 100px;
    height: 100px;
}

Propiedades de Visibilidad

Existen varias formas de controlar la visibilidad de un elemento en CSS, cada una con comportamientos diferentes en cuanto al espacio ocupado, la interactividad y la accesibilidad.

visibility: visible / hidden

  • visible: Estado normal, el elemento es visible (valor predeterminado)
  • hidden: Oculta el elemento pero mantiene su espacio en el layout
  • El elemento no es interactivo cuando está oculto
  • Los lectores de pantalla generalmente ignoran elementos con visibility: hidden
.elemento-visible {
    visibility: visible; /* Valor predeterminado */
}

.elemento-oculto {
    visibility: hidden;
    /* El elemento sigue ocup  /* Valor predeterminado */
}

.elemento-oculto {
    visibility: hidden;
    /* El elemento sigue ocupando espacio en el layout */
}

opacity: 0 - 1

  • Controla la transparencia de un elemento
  • opacity: 1: Completamente visible (valor predeterminado)
  • opacity: 0: Completamente transparente (invisible)
  • Mantiene el espacio del elemento en el layout
  • El elemento sigue siendo interactivo incluso cuando es transparente
  • Permite valores intermedios para transparencia parcial
.elemento-visible {
    opacity: 1; /* Completamente visible */
}

.elemento-semi-transparente {
    opacity: 0.5; /* 50% transparente */
}

.elemento-invisible {
    opacity: 0; /* Completamente transparente */
}

Comparación: display: none vs. visibility: hidden

Aunque ambas propiedades pueden ocultar elementos, tienen diferencias importantes en su comportamiento que afectan al layout, la interactividad y la accesibilidad.

Combinando Propiedades

A menudo, la mejor solución es combinar estas propiedades para lograr el comportamiento deseado:

/* Para transiciones suaves al mostrar/ocultar */
.elemento {
    opacity: 1;
    visibility: visible;
    transition: opacity 0.3s ease;
}

.elemento.oculto {
    opacity: 0;
    visibility: hidden;
    transition: opacity 0.3s ease, visibility 0s linear 0.3s;
}

Ejemplo: Modal con transición suave

.modal {
    opacity: 0;
    visibility: hidden;
    transition: opacity 0.3s, visibility 0s 0.3s;
}

.modal.active {
    opacity: 1;
    visibility: visible;
    transition: opacity 0.3s;
}

Ejemplo: Menú desplegable responsivo

/* En pantallas pequeñas */
@media (max-width: 768px) {
    .menu {
        display: none;
    }
}

/* En pantallas grandes */
@media (min-width: 769px) {
    .menu {
        display: block;
    }
}

Importante: La elección entre display: none, visibility: hidden y opacity: 0 debe basarse en el comportamiento específico que necesitas para tu interfaz de usuario, considerando el layout, la interactividad, la accesibilidad y las animaciones.

Cuándo usar cada propiedad

                
                
    
                    
  • display: block/inline/inline-block: Para controlar el flujo y layout básico de los elementos.
    • 
                    
  • display: flex/grid: Para layouts más complejos y responsivos.
    • 
                    
  • display: none: Cuando necesitas eliminar completamente un elemento del layout.
    • 
                    
  • visibility: hidden: Cuando necesitas ocultar un elemento pero mantener su espacio.
    •

Consideraciones de Accesibilidad

                
                
    
                    
  • Los elementos con display: none o visibility: hidden generalmente son ignorados por los lectores de pantalla.
    • 
                    
  • Los elementos con opacity: 0 siguen siendo accesibles para los lectores de pantalla, lo que puede ser confuso si no deberían serlo.
    • 
                    
  • Para contenido que debe estar disponible solo para lectores de pantalla, usa técnicas como position: absolute; left: -9999px; en lugar de ocultar con display: none.
    • 
                    
  • Asegúrate de que los elementos interactivos ocultos no puedan recibir el foco del teclado si no son visibles.
