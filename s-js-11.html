Programación Orientada a Objetos

La Programación Orientada a Objetos (POO) es un paradigma de programación que organiza el código en "objetos" que representan entidades del mundo real. Cada objeto tiene propiedades (datos) y métodos (comportamientos), permitiendo crear código más modular, reutilizable y fácil de mantener.

Modular
Reutilizable
Escalable

¿Por qué usar POO?

Organización

La POO permite organizar el código en estructuras lógicas que representan entidades reales, haciendo que el código sea más intuitivo y fácil de entender.

Reutilización

Mediante la herencia y la composición, puedes reutilizar código existente, evitando duplicación y facilitando el mantenimiento del proyecto.

Encapsulación

Oculta la complejidad interna y expone solo lo necesario, protegiendo los datos y reduciendo la posibilidad de errores accidentales.

Mantenimiento

Los cambios en una clase afectan solo a esa clase, haciendo que el código sea más fácil de mantener y actualizar sin romper otras partes del sistema.

Escalabilidad

La POO facilita agregar nuevas funcionalidades sin modificar el código existente, permitiendo que tu aplicación crezca de forma ordenada.

Colaboración

Múltiples desarrolladores pueden trabajar en diferentes clases simultáneamente, mejorando la productividad del equipo y reduciendo conflictos.

Encapsulación

La encapsulación es el proceso de ocultar los detalles internos de un objeto y exponer solo lo necesario. Protege los datos al hacer que las propiedades sean privadas y accesibles solo mediante métodos públicos (getters y setters).

Analogía del Mundo Real:

Un cajero automático encapsula su funcionamiento interno. No necesitas saber cómo funciona por dentro; solo usas los botones (interfaz pública) para realizar operaciones. 

class CuentaBancaria {
  // Propiedad privada (con #)
  #saldo = 0;

  constructor(saldoInicial) {
    this.#saldo = saldoInicial;
  }

  // Método público (getter)
  getSaldo() {
    return this.#saldo;
  }

  // Método público
  depositar(cantidad) {
    if (cantidad > 0) {
      this.#saldo += cantidad;
      return true;
    }
    return false;
  }
}

const cuenta = new CuentaBancaria(1000);
cuenta.depositar(500);
console.log(cuenta.getSaldo()); // 1500
// cuenta.#saldo; // ❌ Error: propiedad privada

Abstracción

La abstracción consiste en mostrar solo la información esencial y ocultar los detalles complejos de implementación. Permite trabajar con conceptos de alto nivel sin preocuparse por los detalles internos.

Analogía del Mundo Real:

Cuando conduces un auto, solo necesitas saber usar el volante, pedales y palanca de cambios. No necesitas entender cómo funciona el motor internamente. 

class ReproductorMusica {
  #archivoActual;
  #volumen = 50;

  // Interfaz simple para el usuario
  reproducir(cancion) {
    this.#archivoActual = cancion;
    this.#cargarArchivo();
    this.#decodificarAudio();
    this.#reproducirAudio();
    console.log('▶️ Reproduciendo: ' + cancion);
  }

  pausar() {
    console.log('⏸️ Pausado');
  }

  // Métodos privados (complejidad oculta)
  #cargarArchivo() { /* lógica compleja */ }
  #decodificarAudio() { /* lógica compleja */ }
  #reproducirAudio() { /* lógica compleja */ }
}

const player = new ReproductorMusica();
player.reproducir('cancion.mp3'); // Simple y claro

Herencia

La herencia permite crear nuevas clases basadas en clases existentes, heredando sus propiedades y métodos. Esto facilita la reutilización de código y la creación de jerarquías de clases.

Analogía del Mundo Real:

Un "Perro" y un "Gato" heredan características comunes de "Animal" (respirar, comer), pero cada uno tiene sus propias características específicas (ladrar, maullar). 

// Clase base (padre)
class Animal {
  constructor(nombre) {
    this.nombre = nombre;
  }

  comer() {
    console.log('Comiendo...');
  }
}

// Clase derivada (hija)
class Perro extends Animal {
  constructor(nombre, raza) {
    super(nombre); // Llama al constructor padre
    this.raza = raza;
  }

  ladrar() {
    console.log('🐕 ¡Guau guau!');
  }
}

const miPerro = new Perro('Rex', 'Labrador');
miPerro.comer();  // Heredado de Animal
miPerro.ladrar(); // Específico de Perro

Polimorfismo

El polimorfismo permite que objetos de diferentes clases respondan de manera diferente al mismo método. Una misma interfaz puede tener múltiples implementaciones según el contexto.

Analogía del Mundo Real:

El botón "Play" funciona diferente en un reproductor de música, un videojuego o un reproductor de video, pero todos responden al mismo comando básico. 

class Figura {
  calcularArea() {
    throw new Error('Método debe ser implementado');
  }
}

class Circulo extends Figura {
  constructor(radio) {
    super();
    this.radio = radio;
  }

  calcularArea() {
    return Math.PI * this.radio ** 2;
  }
}

class Rectangulo extends Figura {
  constructor(ancho, alto) {
    super();
    this.ancho = ancho;
    this.alto = alto;
  }

  calcularArea() {
    return this.ancho * this.alto;
  }
}

// Mismo método, diferentes comportamientos
const figuras = [
  new Circulo(5),
  new Rectangulo(4, 6)
];

figuras.forEach(f => console.log(f.calcularArea()));

Clases en JavaScript

Las clases son plantillas para crear objetos con propiedades y métodos predefinidos

Estructura Básica de una Clase

Componentes de una Clase:

  • •
    Constructor: Método especial que inicializa la clase
  • •
    Propiedades: Variables que pertenecen al objeto
  • •
    Métodos: Funciones que pertenecen al objeto
  • •
    Getters/Setters: Acceso controlado a propiedades
  • •
    Métodos Estáticos: Pertenecen a la clase, no a instancias 
class Persona {
  // Propiedad privada
  #edad;

  // Constructor
  constructor(nombre, edad) {
    this.nombre = nombre;
    this.#edad = edad;
  }

  // Getter
  get edad() {
    return this.#edad;
  }

  // Setter
  set edad(valor) {
    if (valor > 0) {
      this.#edad = valor;
    }
  }

  // Método
  saludar() {
    console.log('Hola, soy ' + this.nombre);
  }

  // Método estático
  static crear(nombre, edad) {
    return new Persona(nombre, edad);
  }
}

// Uso
const p1 = new Persona('Ana', 25);
const p2 = Persona.crear('Luis', 30);

Constructor

El constructor es un método especial que se ejecuta automáticamente al crear una instancia de la clase. Se usa para inicializar propiedades. 

class Producto {
  constructor(nombre, precio) {
    this.nombre = nombre;
    this.precio = precio;
    this.id = Date.now();
    console.log('Producto creado');
  }
}

const laptop = new Producto('Laptop', 999);
// "Producto creado"

Métodos Estáticos

Los métodos estáticos pertenecen a la clase, no a las instancias. Se usan para funcionalidades que no requieren datos de instancia. 

class Matematica {
  static sumar(a, b) {
    return a + b;
  }

  static multiplicar(a, b) {
    return a * b;
  }
}

// No necesitas crear instancia
console.log(Matematica.sumar(5, 3)); // 8

Getters y Setters

Getters y Setters permiten controlar el acceso y modification de propiedades, añadiendo validación o lógica adicional. 

class Temperatura {
  #celsius = 0;

  get fahrenheit() {
    return (this.#celsius * 9/5) + 32;
  }

  set celsius(valor) {
    if (valor < -273.15) {
      throw new Error('Temp imposible');
    }
    this.#celsius = valor;
  }
}

const temp = new Temperatura();
temp.celsius = 25;
console.log(temp.fahrenheit); // 77

Propiedades Privadas

Las propiedades privadas (con #) solo son accesibles dentro de la clase, implementando verdadera encapsulación. 

class Usuario {
  #contraseña;

  constructor(nombre, pass) {
    this.nombre = nombre;
    this.#contraseña = this.#hash(pass);
  }

  #hash(str) {
    return 'hashed_' + str;
  }

  verificar(pass) {
    return this.#contraseña === this.#hash(pass);
  }
}

const user = new Usuario('Ana', '1234');
// user.#contraseña; // ❌ Error

Implementación de Herencia

// Clase padre
class Persona {
  constructor(nombre, edad) {
    this.nombre = nombre;
    this.edad = edad;
  }

  saludar() {
    return 'Hola, soy ' + this.nombre;
  }
}

// Clase hija
class Estudiante extends Persona {
  constructor(nombre, edad, carrera) {
    super(nombre, edad); // Llama al padre
    this.carrera = carrera;
  }

  estudiar() {
    return 'Estudiando ' + this.carrera;
  }

  // Sobrescribe método del padre
  saludar() {
    return super.saludar() + ', estudio ' + this.carrera;
  }
}

const est = new Estudiante('Ana', 20, 'Ing. Software');
console.log(est.saludar());
console.log(est.estudiar());

Método super()

La palabra clave super se usa para:

  • 1. Llamar al constructor de la clase padre
  • 2. Acceder a métodos de la clase padre 
class Empleado extends Persona {
  constructor(nombre, edad, salario) {
    // 1. Llamar constructor padre
    super(nombre, edad);
    this.salario = salario;
  }

  presentarse() {
    // 2. Llamar método padre
    const saludo = super.saludar();
    return saludo + ' y gano $' + this.salario;
  }
}

const emp = new Empleado('Carlos', 30, 50000);
console.log(emp.presentarse());

Importante: Debes llamar a super() antes de usar this en el constructor de una clase hija.

Sistema Bancario

class CuentaBancaria {
  #saldo = 0;
  #transacciones = [];

  constructor(titular, saldoInicial) {
    this.titular = titular;
    this.#saldo = saldoInicial;
    this.#registrar('Apertura', saldoInicial);
  }

  depositar(monto) {
    this.#saldo += monto;
    this.#registrar('Depósito', monto);
  }

  retirar(monto) {
    if (this.#saldo >= monto) {
      this.#saldo -= monto;
      this.#registrar('Retiro', monto);
      return true;
    }
    return false;
  }

  #registrar(tipo, monto) {
    this.#transacciones.push({
      tipo, monto,
      fecha: new Date()
    });
  }

  get saldo() {
    return this.#saldo;
  }
}

const cuenta = new CuentaBancaria('Ana', 1000);
cuenta.depositar(500);
cuenta.retirar(200);
console.log(cuenta.saldo); // 1300

Sistema de Figuras Geométricas (Polimorfismo)

class Figura {
  constructor(color) {
    this.color = color;
  }

  calcularArea() {
    throw new Error('Método debe ser implementado');
  }

  describir() {
    return 'Figura de color ' + this.color;
  }
}

class Circulo extends Figura {
  constructor(color, radio) {
    super(color);
    this.radio = radio;
  }

  calcularArea() {
    return Math.PI * this.radio ** 2;
  }

  describir() {
    return super.describir() + ' - Círculo con radio ' + this.radio;
  }
}

class Rectangulo extends Figura {
  constructor(color, ancho, alto) {
    super(color);
    this.ancho = ancho;
    this.alto = alto;
  }

  calcularArea() {
    return this.ancho * this.alto;
  }

  describir() {
    return super.describir() + ' - Rectángulo ' + this.ancho + 'x' + this.alto;
  }
}

class Triangulo extends Figura {
  constructor(color, base, altura) {
    super(color);
    this.base = base;
    this.altura = altura;
  }

  calcularArea() {
    return (this.base * this.altura) / 2;
  }

  describir() {
    return super.describir() + ' - Triángulo base:' + this.base + ' altura:' + this.altura;
  }
}

// Polimorfismo en acción: mismo método, diferentes comportamientos
const figuras = [
  new Circulo('rojo', 5),
  new Rectangulo('azul', 4, 6),
  new Triangulo('verde', 5, 8)
];

figuras.forEach(figura => {
  console.log(figura.describir());
  console.log('Área: ' + figura.calcularArea().toFixed(2));
});

Ejemplo completo de polimorfismo: diferentes figuras implementan el mismo método calcularArea() de manera diferente, pero todas pueden usarse de forma intercambiable.
