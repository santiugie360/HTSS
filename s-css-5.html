Posicionamiento de Elementos

Aprende a controlar la posición de los elementos en CSS con las propiedades position y z-index

Introducción al Posicionamiento

El posicionamiento en CSS permite controlar cómo se ubican los elementos en la página. La propiedad position determina el método de posicionamiento utilizado para un elemento.

Tipos de Posicionamiento

CSS ofrece cinco valores principales para la propiedad position:

  • static: Posicionamiento normal (predeterminado)
  • relative: Relativo a su posición normal
  • absolute: Relativo al ancestro posicionado más cercano
  • fixed: Relativo a la ventana del navegador
  • sticky: Híbrido entre relative y fixed

Propiedades de Desplazamiento

Una vez que un elemento tiene un valor de position distinto de static, puedes usar estas propiedades para ajustar su posición:

  • top: Distancia desde el borde superior
  • right: Distancia desde el borde derecho
  • bottom: Distancia desde el borde inferior
  • left: Distancia desde el borde izquierdo
  • z-index: Controla el orden de apilamiento

Flujo Normal del Documento

Por defecto, los elementos HTML siguen el "flujo normal" del documento:

  • Los elementos de bloque (como div, p, h1) ocupan todo el ancho disponible y crean una nueva línea.
  • Los elementos en línea (como span, a, strong) solo ocupan el espacio necesario y no crean nuevas líneas.
  • Cada elemento se coloca después del anterior, siguiendo el orden del código HTML.

El posicionamiento CSS permite alterar este flujo normal para crear diseños más complejos y dinámicos.

Position: Static

El valor static es el posicionamiento predeterminado para todos los elementos HTML. Un elemento con position: static sigue el flujo normal del documento y no se ve afectado por las propiedades top, right, bottom, left o z-index.

Características

  • Es el valor predeterminado para todos los elementos HTML.
  • El elemento sigue el flujo normal del documento.
  • Las propiedades de desplazamiento (top, right, bottom, left) no tienen efecto.
  • La propiedad z-index no tiene efecto.
  • No se puede usar como referencia para elementos con position: absolute.
.elemento {
    position: static;
    /* Las siguientes propiedades no tendrán efecto */
    top: 20px;
    left: 30px;
    z-index: 10;
}

¿Cuándo usar position: static?

Casos de uso:

  • Para la mayoría de los elementos en un diseño normal.
  • Cuando quieres que un elemento siga el flujo natural del documento.
  • Para "resetear" el posicionamiento de un elemento que podría haber heredado otro tipo de posicionamiento.

Consideraciones:

  • Es el valor predeterminado, por lo que generalmente no es necesario especificarlo explícitamente.
  • No proporciona control sobre la posición exacta de un elemento.
  • Para diseños complejos o elementos que necesitan superponerse, necesitarás otros valores de position.

Nota: Aunque position: static es el valor predeterminado, a veces es útil establecerlo explícitamente para anular otros estilos que podrían estar aplicando un posicionamiento diferente.

Position: Relative

El valor relative posiciona un elemento en relación con su posición normal en el flujo del documento. El elemento mantiene su espacio original en el flujo, pero puede desplazarse usando las propiedades top, right, bottom y left.

Características

  • El elemento mantiene su espacio original en el flujo del documento.
  • Se puede desplazar usando top, right, bottom y left.
  • El desplazamiento es relativo a su posición original.
  • Puede crear superposiciones con otros elementos.
  • Establece un contexto de posicionamiento para elementos descendientes con position: absolute.
.elemento {
    position: relative;
    top: 20px;    /* Desplaza 20px hacia abajo */
    left: 30px;   /* Desplaza 30px hacia la derecha */
}

Position: Absolute

El valor absolute posiciona un elemento en relación con su ancestro posicionado más cercano (cualquier elemento cuya posición no sea static). Si no existe tal ancestro, se posiciona en relación al documento HTML.

Características

  • El elemento se elimina del flujo normal del documento.
  • No deja espacio para el elemento en la página.
  • Se posiciona en relación a su ancestro posicionado más cercano.
  • Si no hay ancestros posicionados, se posiciona en relación al documento HTML.
  • Puede superponerse a otros elementos.
.contenedor {
    position: relative;
    height: 200px;
}

.elemento {
    position: absolute;
    top: 20px;
    left: 30px;
    /* Se posiciona 20px desde arriba y 30px desde la izquierda
       del contenedor posicionado más cercano */
}

Position: Fixed

El valor fixed posiciona un elemento en relación con la ventana del navegador. El elemento permanece en la misma posición incluso cuando se desplaza la página.

Características

  • El elemento se elimina del flujo normal del documento.
  • Se posiciona en relación a la ventana del navegador.
  • Permanece en la misma posición incluso al desplazarse.
  • No se ve afectado por el desplazamiento de la página.
  • Puede superponerse a otros elementos.
.elemento {
    position: fixed;
    top: 20px;
    right: 30px;
    /* Se posiciona 20px desde arriba y 30px desde la derecha
       de la ventana del navegador */
}

Position: Sticky

El valor sticky es un híbrido entre relative y fixed. El elemento se comporta como relative hasta que cruza un punto de desplazamiento especificado, momento en el que se comporta como fixed.

Características

  • Inicialmente se comporta como position: relative.
  • Se convierte en position: fixed cuando alcanza un umbral de desplazamiento.
  • El umbral se define con top, right, bottom o left.
  • Permanece "pegado" dentro de su contenedor padre.
  • Requiere al menos una propiedad de desplazamiento (top, right, bottom, left).
.elemento {
    position: sticky;
    top: 20px;
    /* El elemento se "pegará" a 20px del borde superior
       cuando se desplace más allá de ese punto */
}

Z-Index

La propiedad z-index controla el orden de apilamiento de elementos posicionados que se superponen. Un elemento con un valor de z-index mayor aparecerá encima de un elemento con un valor menor.

Cómo Funciona

  • Solo afecta a elementos posicionados (no static).
  • Acepta valores enteros (positivos, negativos o cero).
  • Un valor mayor coloca el elemento más arriba en la pila.
  • Sin z-index, los elementos se apilan en el orden en que aparecen en el HTML.
  • Los elementos con z-index negativo aparecen detrás de los elementos sin z-index.
.elemento-atras {
    position: absolute;
    z-index: 1;
}

.elemento-medio {
    position: absolute;
    z-index: 2;
}

.elemento-frente {
    position: absolute;
    z-index: 3;
}

Ejemplo Visual

Los elementos con valores de z-index más altos aparecen encima de los elementos con valores más bajos.

Contextos de Apilamiento

¿Qué son los contextos de apilamiento?

Un contexto de apilamiento es una estructura conceptual tridimensional donde los elementos se apilan uno encima de otro. Cada contexto de apilamiento es independiente de los demás.

Un nuevo contexto de apilamiento se crea cuando:

  • Un elemento tiene position distinto de static y un z-index distinto de auto
  • Un elemento tiene opacity menor que 1
  • Un elemento tiene transform, filter, perspective, clip-path, mask, etc.

Ejemplo de Contextos de Apilamiento:

Padre (z-index: 10)
Hijo (z-index: 5)
Elemento (z-index: 5)

Aunque el elemento hijo azul tiene z-index: 5, aparece encima del elemento verde con z-index: 5 porque su padre (rojo) tiene z-index: 10, creando un contexto de apilamiento separado.

Buena práctica: Usa valores de z-index con incrementos (como 10, 20, 30) en lugar de valores consecutivos (1, 2, 3) para facilitar la inserción de nuevos elementos en el futuro sin tener que redefinir todos los valores.
