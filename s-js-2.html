Sintaxis y Tipos de Datos

Aprende los fundamentos de la sintaxis de JavaScript y sus tipos de datos

Variables en JavaScript

Las variables son contenedores para almacenar datos. En JavaScript, existen tres formas de declarar variables: var, let y const, cada una con diferentes características y alcances.

var

La forma tradicional de declarar variables en JavaScript. Tiene alcance de función o global, pero no de bloque.

var edad = 25;
var nombre = "Juan";
var esEstudiante = true;

// Redeclaración permitida
var edad = 26;

// Reasignación permitida
edad = 27;
  • Alcance de función o global
  • Se puede redeclarar
  • Se puede reasignar
  • Se eleva (hoisting) con valor undefined

let

Introducida en ES6 (2015), tiene alcance de bloque y es la forma recomendada para declarar variables que cambiarán de valor.

let edad = 25;
let nombre = "Juan";
let esEstudiante = true;

// Redeclaración NO permitida
// let edad = 26; // Error

// Reasignación permitida
edad = 27;

// Alcance de bloque
if (true) {
  let x = 10; // Solo existe dentro del bloque
}
// console.log(x); // Error
  • Alcance de bloque
  • No se puede redeclarar en el mismo ámbito
  • Se puede reasignar
  • Se eleva (hoisting) sin inicializar (zona muerta temporal)

const

También introducida en ES6, se utiliza para declarar constantes cuyo valor no cambiará. Tiene alcance de bloque.

const PI = 3.14159;
const NOMBRE = "Juan";
const ES_ESTUDIANTE = true;

// Redeclaración NO permitida
// const PI = 3.14; // Error

// Reasignación NO permitida
// PI = 3.14; // Error

// Pero los objetos y arrays pueden modificarse
const persona = { nombre: "Ana" };
persona.nombre = "Luis"; // Válido
persona.edad = 30; // Válido
  • Alcance de bloque
  • No se puede redeclarar
  • No se puede reasignar
  • Los objetos y arrays pueden modificarse internamente
  • Se eleva (hoisting) sin inicializar (zona muerta temporal)

Buena práctica: En JavaScript moderno, se recomienda usar const por defecto y cambiar a let solo cuando sea necesario reasignar la variable. Evita usar var en código nuevo.

Tipos de Datos Primitivos

JavaScript tiene siete tipos de datos primitivos, que son valores inmutables que se pasan por valor. Estos son los bloques de construcción básicos para cualquier programa JavaScript.

String

string

Representa texto. Se puede definir con comillas simples, dobles o backticks (template literals).

// Formas de crear strings
let nombre = 'Ana';
let apellido = "García";
let saludo = `Hola, ${nombre}!`;  // Template literal

// Métodos comunes
nombre.length;         // 3
nombre.toUpperCase();  // "ANA"
nombre.charAt(0);      // "A"
nombre.indexOf('n');   // 1
nombre.slice(1);       // "na"

Los strings son inmutables, lo que significa que los métodos que parecen modificarlos en realidad devuelven nuevos strings.

Number

number

Representa números, tanto enteros como de punto flotante. JavaScript no distingue entre ellos.

// Números enteros y decimales
let entero = 42;
let decimal = 3.14159;
let negativo = -10;
let exponente = 2.5e6;  // 2500000

// Valores especiales
let infinito = Infinity;
let noEsNumero = NaN;  // Not a Number

// Métodos y propiedades
Number.MAX_VALUE;      // El mayor número posible
(123.456).toFixed(2);  // "123.46"
parseInt("42");        // 42
parseFloat("3.14");    // 3.14

JavaScript utiliza el estándar IEEE 754 para números, lo que puede llevar a imprecisiones en cálculos con decimales (ej: 0.1 + 0.2 !== 0.3).

Boolean

boolean

Representa un valor lógico: verdadero o falso. Se utiliza para tomar decisiones en el código.

// Valores booleanos
let verdadero = true;
let falso = false;

// Operaciones lógicas
let and = true && false;  // false
let or = true || false;   // true
let not = !true;          // false

// Expresiones que devuelven booleanos
let mayorQue = 5 > 3;     // true
let igualdad = 5 === "5"; // false
let desigualdad = 5 !== 4; // true

Los booleanos son fundamentales para las estructuras de control como if, while y for, que ejecutan código basado en condiciones.

Undefined

undefined

Representa una variable que ha sido declarada pero no inicializada, o una propiedad que no existe.

// Variables sin inicializar
let sinValor;
console.log(sinValor);  // undefined

// Parámetros de función no proporcionados
function saludar(nombre) {
  console.log(nombre);  // undefined si no se pasa
}
saludar();

// Propiedades inexistentes
let obj = {};
console.log(obj.propiedad);  // undefined

// Comprobación
typeof sinValor === "undefined";  // true

Undefined indica la ausencia de valor asignado, a diferencia de null que indica la ausencia intencional de valor.

Null

null

Representa la ausencia intencional de cualquier valor u objeto. Es un valor asignado explícitamente.

// Asignación explícita de null
let valorNulo = null;

// Uso común para inicializar variables
let usuario = null;  // Aún no hay usuario

// Comprobación
valorNulo === null;  // true

// Curiosidad
console.log(typeof null);  // "object" (esto es un error histórico)

Null es un valor primitivo, pero curiosamente typeof null devuelve "object", lo cual es considerado un error en el diseño de JavaScript que no se ha corregido por razones de compatibilidad.

Symbol

symbol

Introducido en ES6, representa un identificador único e inmutable. Se utiliza principalmente como claves de propiedades de objetos.

// Creación de símbolos
let id = Symbol();
let id2 = Symbol('id');  // Con descripción

// Símbolos son siempre únicos
Symbol('id') === Symbol('id');  // false

// Uso como claves de propiedades
let obj = {};
obj[id] = 'Valor secreto';

// Símbolos no son enumerables
console.log(Object.keys(obj));  // []

// Símbolos globales
let globalId = Symbol.for('id');
let sameId = Symbol.for('id');
globalId === sameId;  // true

Los símbolos son útiles para añadir propiedades a objetos que no entren en conflicto con otras propiedades, incluso si tienen el mismo nombre.

BigInt

bigint

Introducido en ES2020, representa números enteros de precisión arbitraria, superando las limitaciones de Number.

// Creación de BigInt
let numeroGrande = 9007199254740991n;  // Con sufijo n
let otroGrande = BigInt("9007199254740991");

// Operaciones
let suma = numeroGrande + 1n;  // 9007199254740992n
let producto = numeroGrande * 2n;  // 18014398509481982n

// No se pueden mezclar con Number
// numeroGrande + 1;  // Error

// Conversión
Number(numeroGrande);  // 9007199254740991
BigInt(Number.MAX_SAFE_INTEGER);  // 9007199254740991n

BigInt es útil para trabajar con números enteros muy grandes, como identificadores de bases de datos, criptografía o cálculos científicos de alta precisión.

Verificación de tipos

JavaScript proporciona varias formas de verificar el tipo de un valor:

Operador typeof

typeof "Hola";        // "string"
typeof 42;            // "number"
typeof true;          // "boolean"
typeof undefined;     // "undefined"
typeof null;          // "object" (error histórico)
typeof Symbol();      // "symbol"
typeof 42n;           // "bigint"
typeof {};            // "object"
typeof [];            // "object"
typeof function(){};  // "function"

El operador typeof devuelve una cadena que indica el tipo del operando. Tiene algunas peculiaridades, como devolver "object" para null y arrays.

Métodos de verificación

// Para arrays
Array.isArray([]);  // true
Array.isArray({});  // false

// Para NaN
isNaN(NaN);        // true
isNaN("texto");    // true (convierte a número primero)
Number.isNaN(NaN); // true
Number.isNaN("texto"); // false (más estricto)

// Para números finitos
isFinite(42);      // true
isFinite(Infinity); // false
Number.isFinite(42); // true

Estos métodos proporcionan verificaciones más precisas para casos específicos donde typeof puede ser ambiguo o engañoso.

Consejo: En JavaScript moderno, se recomienda usar Array.isArray() para verificar arrays y Number.isNaN() en lugar de isNaN() para verificaciones más precisas.

Tipos de datos no primitivos

Además de los tipos primitivos, JavaScript tiene tipos de datos no primitivos (objetos) que se pasan por referencia:

Object

object 
// Objetos literales
let persona = {
  nombre: "Ana",
  edad: 28,
  saludar: function() {
    return `Hola, soy ${this.nombre}`;
  }
};

// Acceso a propiedades
persona.nombre;  // "Ana"
persona["edad"]; // 28

// Métodos
persona.saludar();  // "Hola, soy Ana"

// Añadir/modificar propiedades
persona.ciudad = "Madrid";
persona.edad = 29;

Array

object 
// Arrays
let colores = ["rojo", "verde", "azul"];
let mixto = [1, "dos", true, null, {a: 1}];

// Acceso a elementos
colores[0];  // "rojo"
colores.length;  // 3

// Métodos comunes
colores.push("amarillo");  // Añade al final
colores.pop();  // Elimina del final
colores.unshift("negro");  // Añade al inicio
colores.shift();  // Elimina del inicio
colores.slice(1, 3);  // Subarray
colores.splice(1, 1, "púrpura");  // Reemplaza

Function

function 
// Declaración de función
function sumar(a, b) {
  return a + b;
}

// Expresión de función
const restar = function(a, b) {
  return a - b;
};

// Arrow function (ES6)
const multiplicar = (a, b) => a * b;

// Funciones como objetos
sumar.descripcion = "Función que suma dos números";
console.log(sumar.descripcion);

// Funciones como argumentos
[1, 2, 3].map(x => x * 2);  // [2, 4, 6]

Otros objetos

// RegExp

mapa.set("clave", "valor");

Conversión de Tipos

JavaScript es un lenguaje de tipado dinámico, lo que significa que las variables pueden cambiar de tipo. La conversión de tipos puede ser explícita (cuando el programador la realiza intencionalmente) o implícita (cuando JavaScript la realiza automáticamente).

Conversión explícita (Type Casting)

Es cuando convertimos un tipo a otro de forma intencional usando funciones o métodos específicos.

// String a Number
Number("42");        // 42
parseInt("42");      // 42
parseFloat("3.14");  // 3.14
+"42";               // 42 (operador unario +)

// Number a String
String(42);          // "42"
(42).toString();     // "42"
42 + "";             // "42" (concatenación)

// A Boolean
Boolean(1);          // true
Boolean(0);          // false
Boolean("");         // false
Boolean("texto");    // true
!!"texto";           // true (doble negación)

// A Number
Number(true);        // 1
Number(false);       // 0

// A Object
Object(42);          // Number {42}
Object("texto");     // String {"texto"}

// De Object a primitivo
Number([1]);         // 1
Number([1,2]);       // NaN
String([1,2]);       // "1,2"

La conversión explícita es más clara y predecible, por lo que se recomienda usarla cuando sea posible para evitar comportamientos inesperados.

Conversión implícita (Coerción)

Ocurre cuando JavaScript convierte automáticamente un tipo a otro durante la ejecución de operaciones.

// Operador +
"5" + 3;             // "53" (string concatenation)
3 + "5";             // "35" (string concatenation)
true + 1;            // 2 (true se convierte a 1)
null + 1;            // 1 (null se convierte a 0)
undefined + 1;       // NaN (undefined se convierte a NaN)

// Otros operadores aritméticos
"5" - 3;             // 2 (string se convierte a number)
"5" * 3;             // 15 (string se convierte a number)
"5" / "2";           // 2.5 (ambos se convierten a number)

// Operadores de comparación
"5" == 5;            // true (comparación con coerción)
"5" === 5;           // false (comparación estricta, sin coerción)
0 == false;          // true (coerción)
0 === false;         // false (sin coerción)
null == undefined;   // true (coerción)
null === undefined;  // false (sin coerción)

// Operadores lógicos
"texto" && 1;        // 1 (ambos son truthy)
"" || "alternativa"; // "alternativa" (el primero es falsy)
!0;                  // true (0 es falsy)

La coerción puede llevar a comportamientos inesperados si no se comprende bien. Por eso se recomienda usar el operador de igualdad estricta (===) en lugar del de igualdad abstracta (==).

Valores Truthy y Falsy

En JavaScript, cualquier valor puede ser convertido a booleano en contextos que requieren un booleano (como condiciones if). Los valores se clasifican como "truthy" (se convierten a true) o "falsy" (se convierten a false).

Valores Falsy

Estos son todos los valores que se convierten a false:

  • false - El valor booleano false
  • 0 - El número cero
  • -0 - El número cero negativo
  • 0n - BigInt cero
  • "", '', `` - String vacío
  • null - Ausencia de valor
  • undefined - Valor no definido
  • NaN - Not a Number
if (0) {
  // Este bloque no se ejecuta
}

if ("") {
  // Este bloque no se ejecuta
}

if (null || undefined) {
  // Este bloque no se ejecuta
}

Valores Truthy

Todos los demás valores se convierten a true, incluyendo:

  • true - El valor booleano true
  • Cualquier número distinto de cero (positivo o negativo)
  • Cualquier string no vacío
  • Objetos (incluso vacíos)
  • Arrays (incluso vacíos)
  • Funciones
  • Símbolos
if (1) {
  // Este bloque se ejecuta
}

if ("texto") {
  // Este bloque se ejecuta
}

if ({} || []) {
  // Este bloque se ejecuta
}

Aplicación práctica: Los valores truthy y falsy son útiles para verificaciones condicionales simplificadas. Por ejemplo, if (valor) en lugar de if (valor !== null && valor !== undefined && valor !== '').

Operadores y Expresiones

Los operadores permiten realizar operaciones con variables y valores. JavaScript proporciona varios tipos de operadores para diferentes propósitos.

Operadores aritméticos

Realizan operaciones matemáticas básicas.

// Operadores básicos
5 + 3;      // 8 (suma)
5 - 3;      // 2 (resta)
5 * 3;      // 15 (multiplicación)
5 / 3;      // 1.6666... (división)
5 % 3;      // 2 (módulo - resto de la división)
5 ** 3;     // 125 (exponenciación - ES2016)

// Incremento y decremento
let x = 5;
x++;        // Post-incremento (devuelve 5, luego x = 6)
++x;        // Pre-incremento (x = 7, devuelve 7)
x--;        // Post-decremento (devuelve 7, luego x = 6)
--x;        // Pre-decremento (x = 5, devuelve 5)

// Operadores de asignación compuesta
x = 5;
x += 3;     // x = x + 3 (x = 8)
x -= 2;     // x = x - 2 (x = 6)
x *= 2;     // x = x * 2 (x = 12)
x /= 4;     // x = x / 4 (x = 3)
x %= 2;     // x = x % 2 (x = 1)
x **= 3;    // x = x ** 3 (x = 1)

Operadores de comparación

Comparan valores y devuelven un resultado booleano.

// Comparaciones básicas
5 > 3;       // true (mayor que)
5 < 3;       // false (menor que)
5 >= 5;      // true (mayor o igual que)
5 <= 4;      // false (menor o igual que)

// Igualdad abstracta (con coerción)
5 == 5;      // true
5 == "5";    // true (string se convierte a number)
0 == false;  // true (false se convierte a 0)
null == undefined;  // true

// Igualdad estricta (sin coerción)
5 === 5;     // true
5 === "5";   // false (tipos diferentes)
0 === false; // false (tipos diferentes)
null === undefined; // false (tipos diferentes)

// Desigualdad
5 != 3;      // true (con coerción)
5 != "5";    // false (con coerción)
5 !== "5";   // true (estricta, sin coerción)

Buena práctica: Utiliza siempre los operadores de igualdad estricta (=== y !==) para evitar conversiones de tipo inesperadas.

Operadores lógicos

Realizan operaciones lógicas y se utilizan frecuentemente en estructuras de control.

// Operadores lógicos básicos
true && true;   // true (AND lógico)
true && false;  // false
true || false;  // true (OR lógico)
false || false; // false
!true;          // false (NOT lógico)
!false;         // true

// Evaluación de cortocircuito
false && cualquierCosa;  // false (no evalúa cualquierCosa)
true || cualquierCosa;   // true (no evalúa cualquierCosa)

// Usos comunes
let edad = 20;
let tienePermiso = true;

if (edad >= 18 && tienePermiso) {
  // Se ejecuta si ambas condiciones son true
}

// Valores por defecto (antes de ES6)
let nombre = nombreUsuario || "Invitado";

// Ejecución condicional
isAdmin && ejecutarFuncionAdmin();

Operadores modernos (ES2020+)

JavaScript ha introducido nuevos operadores en versiones recientes que simplifican tareas comunes.

// Operador de coalescencia nula (??)
// Devuelve el operando derecho cuando el izquierdo es null o undefined
let valor = null ?? "valor por defecto";  // "valor por defecto"
let cero = 0 ?? 42;  // 0 (0 no es null ni undefined)
let vacio = "" ?? "valor por defecto";  // "" (string vacío no es null)

// Operador de encadenamiento opcional (?.)
// Evita errores al acceder a propiedades de objetos potencialmente nulos
let usuario = null;
let nombre = usuario?.nombre;  // undefined (no error)
let subProp = usuario?.direccion?.ciudad;  // undefined (no error)
let metodo = usuario?.saludar?.();  // undefined (no error)

// Asignación lógica (ES2021)
let x;
// Asignación con AND lógico
x &&= 5;  // Equivale a: x && (x = 5)
// Asignación con OR lógico
x ||= 5;  // Equivale a: x || (x = 5)
// Asignación con coalescencia nula
x ??= 5;  // Equivale a: x ?? (x = 5)

Nota: Estos operadores modernos pueden no estar disponibles en navegadores antiguos sin un transpilador como Babel.

Precedencia y asociatividad de operadores

La precedencia determina el orden en que se evalúan los operadores. Los operadores con mayor precedencia se evalúan antes que los de menor precedencia. La asociatividad determina el orden de evaluación cuando hay varios operadores con la misma precedencia.

Precedencia Operador Asociatividad Ejemplo
19 Agrupación ( ... ) n/a (3 + 4) * 5
17 Acceso a miembros . [] Izquierda a derecha objeto.propiedad, array[0]
16 ++ -- ! typeof Derecha a izquierda !false, typeof x
15 ** (exponenciación) Derecha a izquierda 2 ** 3 ** 2 (2 ** 9)
14 * / % Izquierda a derecha 15 * 5 / 3
13 + - Izquierda a derecha 10 + 5 - 3
11 < <= > >= Izquierda a derecha x > y >= z
10 == != === !== Izquierda a derecha x === y !== z
6 && Izquierda a derecha x > 0 && y < 10
5 || Izquierda a derecha x || y || z
3 = += -= *= /= %= Derecha a izquierda x = y = z

Consejo: Cuando tengas dudas sobre la precedencia, usa paréntesis para hacer explícito el orden de evaluación. Esto mejora la legibilidad y evita errores sutiles.
