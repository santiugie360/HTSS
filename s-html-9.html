HTML Estructura del Documento

Todo documento HTML tiene una estructura fundamental que define cómo los navegadores interpretan y muestran el contenido. Desde la declaración DOCTYPE hasta los metadatos en el head, cada parte cumple una función específica para crear páginas web válidas, accesibles y optimizadas.

<!DOCTYPE>
<html>
<head>
<body>

¿Qué es la Estructura del Documento?

Anatomía de un Documento

Un documento HTML válido sigue una estructura específica que comienza con la declaración <!DOCTYPE html>, seguida de la etiqueta <html> que contiene dos secciones principales.

Esta estructura es obligatoria y permite a los navegadores renderizar correctamente el contenido.

Head vs Body

El <head> contiene información sobre el documento (metadatos, títulos, enlaces a CSS) que NO se muestra directamente en la página. El <body> contiene todo el contenido visible.

Separar metadatos de contenido es fundamental para la organización y optimización web.

Metadatos Esenciales

Los metadatos en el <head> incluyen: codificación de caracteres, viewport para responsive design, descripción para SEO, y enlaces a recursos externos como CSS y JavaScript.

Configurar correctamente los metadatos es crucial para SEO, accesibilidad y rendimiento.

Validación y Estándares

La declaración <!DOCTYPE html> asegura que el navegador use el modo de estándares más reciente. Sin ella, los navegadores pueden entrar en "quirks mode" con comportamiento impredecible.

Siempre incluye el DOCTYPE para garantizar compatibilidad y comportamiento consistente.

  <script src="script.js"></script>

Declaración <!DOCTYPE>

La primera línea de cualquier documento HTML

¿Qué es DOCTYPE?

DOCTYPE (Document Type Declaration) le indica al navegador qué versión de HTML estás usando. En HTML5, es simple y corto. 

<!DOCTYPE html>
✓ HTML5 (Actual): <!DOCTYPE html>
✗ HTML 4.01 (Obsoleto): <!DOCTYPE HTML PUBLIC "-//W3C//DTD HTML 4.01//EN"...>
✗ XHTML (Obsoleto): <!DOCTYPE html PUBLIC "-//W3C//DTD XHTML 1.0...>

¿Por qué es Importante?

✓
Modo Estándares

Activa el modo de renderizado moderno y consistente

✓
Evita Quirks Mode

Sin DOCTYPE, el navegador entra en modo compatible (comportamiento antiguo e impredecible)

✓
Validación HTML

Permite que validadores verifiquen tu código correctamente

✓
SEO

Los motores de búsqueda prefieren HTML válido

⚠️ Importante: SIEMPRE incluye <!DOCTYPE html> como la primera línea de tu documento HTML.

Elemento <html>

El elemento raíz que contiene todo el documento

Elemento Raíz

La etiqueta <html> es el contenedor principal de todo el documento HTML. Contiene dos secciones: <head> y <body>. 

<!DOCTYPE html>
<html lang="es">
  <head>
    <!-- Metadatos -->
  </head>
  <body>
    <!-- Contenido -->
  </body>
</html>

Nota: Todo el contenido visible e invisible debe estar dentro de <html>.

Atributo lang

El atributo lang especifica el idioma del documento. Es crucial para accesibilidad y SEO.

<html lang="es"> Español
<html lang="en"> Inglés
<html lang="fr"> Francés
<html lang="pt-BR"> Portugués (Brasil)

Beneficios:

  • ✓
    Lectores de pantalla usan la pronunciación correcta
  • ✓
    Motores de búsqueda indexan en el idioma correcto
  • ✓
    Los navegadores pueden ofrecer traducción automática
  • ✓
    CSS puede aplicar estilos según el idioma

Elemento <head>

Metadatos e información sobre el documento

¿Qué contiene el Head?

El <head> contiene metadatos sobre el documento que NO se muestran en la página. Incluye:

  • •
    Título de la página
  • •
    Codificación de caracteres
  • •
    Meta tags para SEO
  • •
    Enlaces a hojas de estilo CSS
  • •
    Scripts de JavaScript
  • •
    Favicon y otros iconos

Ejemplo Básico:

<head>
  <!-- Obligatorio: Codificación -->
  <meta charset="UTF-8">
  
  <!-- Obligatorio: Título -->
  <title>Mi Página</title>
  
  <!-- Recomendado: Viewport -->
  <meta name="viewport" 
        content="width=device-width, initial-scale=1.0">
  
  <!-- Opcional: CSS -->
  <link rel="stylesheet" href="style.css">
</head>

<title> - Título del Documento

El título aparece en la pestaña del navegador, en resultados de búsqueda, y al compartir en redes sociales. Es obligatorio. 

<title>Mi Sitio Web - Inicio</title>
Mi Sitio Web - Inicio

Contenido de la página...

SEO Tip: Mantén el título entre 50-60 caracteres. Incluye palabras clave importantes al inicio.

<meta charset> - Codificación de Caracteres

Define cómo el navegador interpreta los caracteres del documento. UTF-8 soporta todos los idiomas y caracteres especiales. 

<meta charset="UTF-8">

✓ Con UTF-8: ñ, á, é, í, ó, ú, ü, €, ©, ™, 你好, مرحبا

✗ Sin UTF-8: �, �, �, � (caracteres rotos)

Importante: Debe ser la PRIMERA etiqueta meta en el <head>.

<meta viewport> - Diseño Responsive

Esencial para que tu sitio se vea bien en dispositivos móviles. Controla cómo se escala la página en diferentes tamaños de pantalla. 

<meta name="viewport" 
      content="width=device-width, initial-scale=1.0">

Parámetros del content:

  • width=device-width - Ancho = ancho del dispositivo
  • initial-scale=1.0 - Zoom inicial al 100%
  • maximum-scale=5.0 - Zoom máximo (opcional)
  • user-scalable=yes - Permitir zoom (opcional)

⚠️ Accesibilidad: No uses user-scalable=no, impide que usuarios con baja visión amplíen el contenido.

<link> - Vincular Recursos Externos

Hoja de Estilos CSS:

<link rel="stylesheet" href="styles.css">

Favicon:

<link rel="icon" href="favicon.ico">

Fuentes de Google:

<link rel="preconnect" href="https://fonts.googleapis.com">
<link href="https://fonts.googleapis.com/css2?family=Roboto" 
      rel="stylesheet">

Valores comunes de rel:

stylesheet - Hoja de estilos CSS
icon - Favicon del sitio
canonical - URL canónica (SEO)
alternate - Versión alternativa
preload - Precargar recurso
preconnect - Conectar anticipadamente

Elemento <body>

El contenido visible de tu página web

¿Qué es el Body?

El <body> contiene TODO el contenido visible de la página: texto, imágenes, videos, formularios, etc. 

<body>
  <header>
    <h1>Título Principal</h1>
    <nav>Navegación</nav>
  </header>
  
  <main>
    <article>Contenido</article>
  </main>
  
  <footer>Pie de página</footer>
</body>

Nota: Solo debe haber UN elemento <body> por documento.

Estructura Semántica

Usa elementos semánticos HTML5 para estructurar el body:

<header>

Encabezado de la página o sección

<nav>

Navegación principal

<main>

Contenido principal (solo uno)

<article>

Contenido independiente

<section>

Sección temática

<aside>

Contenido relacionado/sidebar

<footer>

Pie de página

¿Dónde colocar los Scripts?

❌ Mal (en el head):

<head>
  <!-- Bloquea el renderizado -->
  <script src="script.js"></script>
</head>

Problema: El navegador espera a descargar el script antes de mostrar la página.

✓ Bien (antes de cerrar body):

<body>
  <!-- Todo el contenido -->
  
  <!-- Scripts al final -->
  <script src="script.js"></script>
</body>

Ventaja: La página se muestra primero, luego se carga el JavaScript.

Alternativa moderna (con defer):

<head>
  <!-- Se descarga en paralelo, se ejecuta después del HTML -->
  <script src="script.js" defer></script>
</head>

Metadatos Básicos

Información esencial para navegadores, motores de búsqueda y redes sociales

Description - Descripción SEO

Aparece en resultados de búsqueda. Máximo 155-160 caracteres.

<meta name="description" 
      content="Aprende HTML desde cero con ejemplos prácticos y guías completas.">
Mi Sitio Web - Aprende HTML
https://ejemplo.com
Aprende HTML desde cero con ejemplos prácticos y guías completas.

Keywords - Palabras Clave

Palabras clave relevantes (ya no usado por Google, pero otros buscadores sí).

<meta name="keywords" 
      content="HTML, CSS, JavaScript, desarrollo web, tutoriales">

Nota: Google ignora este meta tag, pero otros buscadores aún lo usan.

Author - Autor del Documento

Especifica quién creó el contenido.

<meta name="author" content="Juan Pérez">

Robots - Instrucciones para Rastreadores

Controla cómo los motores de búsqueda indexan tu página.

<!-- Permitir indexación y seguir enlaces -->
<meta name="robots" content="index, follow">

<!-- No indexar -->
<meta name="robots" content="noindex, nofollow">

Open Graph - Compartir en Redes Sociales

Controla cómo se ve tu página al compartirla en Facebook, LinkedIn, etc.

<meta property="og:title" content="Título para Redes">
<meta property="og:description" content="Descripción">
<meta property="og:image" content="imagen.jpg">
<meta property="og:url" content="https://ejemplo.com">

Twitter Card - Compartir en Twitter/X

Específico para Twitter/X, controla la apariencia de las tarjetas.

<meta name="twitter:card" content="summary_large_image">
<meta name="twitter:title" content="Título">
<meta name="twitter:description" content="Descripción">
<meta name="twitter:image" content="imagen.jpg">

Theme Color - Color del Navegador

Define el color de la barra de navegación en móviles.

<meta name="theme-color" content="#f97316">

Así se ve la barra en móvil con theme-color naranja

Canonical URL - URL Canónica

Evita contenido duplicado indicando la URL preferida.

<link rel="canonical" href="https://ejemplo.com/pagina">

SEO: Útil cuando la misma página es accesible desde múltiples URLs.

Ejercicios Prácticos

Resuelve estos ejercicios para consolidar tu aprendizaje

Ejercicio 1: Completa el Documento HTML

Se te proporciona un documento HTML incompleto. Añade los elementos que faltan en el head para que sea válido y optimizado para SEO.

Código Incompleto:

<!DOCTYPE html>
<html lang="es">
<head>
  <title>Mi Blog</title>
  <!-- Falta: meta charset -->
  <!-- Falta: meta viewport -->
  <!-- Falta: meta description -->
</head>
<body>
  <h1>Bienvenido a mi blog</h1>
</body>
</html>

Objetivo: Agrega los 3 meta tags faltantes en el order correcto.

Solución Correcta:

<!DOCTYPE html>
<html lang="es">
<head>
  <!-- PRIMERO: charset -->
  <meta charset="UTF-8">
  
  <!-- SEGUNDO: viewport -->
  <meta name="viewport" 
        content="width=device-width, initial-scale=1.0">
  
  <!-- TERCERO: description -->
  <meta name="description" 
        content="Mi blog personal con artículos sobre desarrollo web">
  
  <title>Mi Blog</title>
</head>
<body>
  <h1>Bienvenido a mi blog</h1>
</body>
</html>

Explicación: El charset debe ser PRIMERO, luego viewport, y la description es opcional pero recomendada para SEO.

Ejercicio 2: Estructura HTML Semántica

Organiza correctamente los elementos semánticos HTML en el body. Identifica qué elemento debería ir en cada sección.

Estructura sin Organizar:

<body>
  <!-- ¿Dónde va cada elemento? -->
  
  Logo y Menú de navegación
  <!-- Elemento: ? -->
  
  Artículo principal del sitio
  <!-- Elemento: ? -->
  
  Información de copyright
  <!-- Elemento: ? -->
  
  Publicidad y enlaces relacionados
  <!-- Elemento: ? -->

</body>

Objetivo: Usa <header>, <main>, <footer>, <aside> correctamente.

Estructura Correcta:

<body>
  <header>
    <!-- Logo y navegación -->
    <nav>...</nav>
  </header>

  <main>
    <!-- Contenido principal -->
    <article>...</article>
  </main>

  <aside>
    <!-- Sidebar/publicidad -->
  </aside>

  <footer>
    <!-- Copyright e info -->
  </footer>
</body>

Explicación: <header> encabezado, <main> contenido principal, <aside> información adicional, <footer> pie de página.
