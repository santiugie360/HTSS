Funciones en JavaScript

Aprende a crear y utilizar funciones para organizar y reutilizar tu código

Declaración y Ejecución de Funciones

Las funciones son bloques de código reutilizables que realizan una tarea específica. Son como pequeñas "máquinas" que pueden recibir datos, procesarlos y devolver un resultado. Las funciones te permiten dividir tu código en partes más pequeñas y manejables.

Declaración de Funciones

La forma más básica de crear una función es usando la palabra clave function, seguida del nombre de la función, paréntesis para los parámetros y llaves para el cuerpo de la función.

// Sintaxis básica
function nombreDeLaFuncion() {
  // Código a ejecutar
}

// Ejemplo: función que muestra un saludo
function saludar() {
  console.log("¡Hola, mundo!");
}

// Ejemplo: función que calcula el área de un rectángulo
function calcularAreaRectangulo(ancho, alto) {
  return ancho * alto;
}

Las funciones pueden contener cualquier código JavaScript válido, incluyendo condicionales, bucles y hasta otras funciones.

Ejecución de Funciones

Para ejecutar (o "llamar") a una función, escribes su nombre seguido de paréntesis. Si la función espera parámetros, los colocas dentro de los paréntesis.

// Llamar a una función sin parámetros
saludar();  // Muestra: ¡Hola, mundo!

// Llamar a una función con parámetros
let area = calcularAreaRectangulo(5, 3);
console.log(area);  // Muestra: 15

// Puedes llamar a una función múltiples veces
saludar();  // Muestra: ¡Hola, mundo!
saludar();  // Muestra: ¡Hola, mundo!

// Puedes usar el resultado en otras operaciones
let areaDoble = calcularAreaRectangulo(5, 3) * 2;
console.log(areaDoble);  // Muestra: 30

Cada vez que llamas a una función, JavaScript ejecuta el código dentro de ella. Esto te permite reutilizar el mismo código sin tener que escribirlo varias veces.

Visualización de una Función

Piensa en una función como una máquina que toma entradas (parámetros), realiza un proceso y produce una salida (valor de retorno):

Entrada (Parámetros):

ancho: 5
alto: 3

Proceso (Cuerpo de la función):

function calcularAreaRectangulo(ancho, alto) {
  return ancho * alto;
}

1. Recibe los parámetros ancho = 5 y alto = 3

2. Multiplica ancho * alto (5 * 3 = 15)

3. Devuelve el resultado con return

Salida (Valor de retorno):

15

Hoisting de Funciones

En JavaScript, las declaraciones de funciones se "elevan" (hoisting) al principio del ámbito, lo que significa que puedes llamar a una función antes de declararla en el código.

// Esto funciona gracias al hoisting
saludar();  // Muestra: ¡Hola!

// La declaración de la función se eleva
function saludar() {
  console.log("¡Hola!");
}

// Pero esto NO funciona con expresiones de función
// saludarExpresion();  // Error: saludarExpresion is not a function

let saludarExpresion = function() {
  console.log("¡Hola desde expresión!");
};

El hoisting solo funciona con declaraciones de funciones, no con expresiones de función o funciones flecha.

Buenas Prácticas

Algunas recomendaciones para escribir funciones efectivas:

  • Usa nombres descriptivos que indiquen lo que hace la función (ej. calcularTotal en lugar de ct)
  • Cada función debería hacer una sola cosa (principio de responsabilidad única)
  • Mantén tus funciones pequeñas y enfocadas
  • Documenta tus funciones con comentarios, especialmente si son complejas
  • Usa valores de retorno en lugar de variables globales
  • Maneja los errores adecuadamente dentro de tus funciones

Consejo: Piensa en las funciones como bloques de construcción. Cada función debería ser como una pieza de LEGO que puedes reutilizar en diferentes partes de tu aplicación.

Parámetros y Valores de Retorno

Los parámetros son los datos que una función necesita para realizar su tarea, mientras que el valor de retorno es el resultado que la función produce. Entender cómo trabajar con ellos es fundamental para crear funciones útiles y flexibles.

Parámetros Básicos

Los parámetros son variables que reciben valores cuando se llama a la función. Actúan como "entradas" para la función.

// Función con dos parámetros
function sumar(a, b) {
  return a + b;
}

// Llamadas a la función con diferentes argumentos
let resultado1 = sumar(5, 3);  // resultado1 = 8
let resultado2 = sumar(10, 20);  // resultado2 = 30

// Los parámetros pueden ser de cualquier tipo
function saludarPersona(nombre, edad) {
  console.log(`Hola ${nombre}, tienes ${edad} años.`);
}

saludarPersona("Ana", 25);  // Muestra: Hola Ana, tienes 25 años.
saludarPersona("Carlos", 30);  // Muestra: Hola Carlos, tienes 30 años.

Los valores que pasas a una función cuando la llamas se denominan "argumentos". Estos argumentos se asignan a los parámetros definidos en la función.

Valores de Retorno

El valor de retorno es el resultado que una función produce y devuelve al código que la llamó. Se especifica con la palabra clave return.

// Función que devuelve un valor
function multiplicar(a, b) {
  return a * b;  // Devuelve el producto de a y b
}

let producto = multiplicar(4, 5);  // producto = 20

// Puedes usar el valor devuelto directamente
console.log(multiplicar(3, 7));  // Muestra: 21

// Una función puede devolver diferentes valores
function esPositivo(numero) {
  if (numero > 0) {
    return true;
  } else {
    return false;
  }
}

// También puedes devolver objetos o arrays
function crearPersona(nombre, edad) {
  return {
    nombre: nombre,
    edad: edad,
    esMayorDeEdad: edad >= 18
  };
}

let persona = crearPersona("Laura", 22);
console.log(persona.esMayorDeEdad);  // Muestra: true

Cuando JavaScript encuentra una instrucción return, la función termina inmediatamente y devuelve el valor especificado. Si no hay un return explícito, la función devuelve undefined.

Parámetros por Defecto

Puedes asignar valores por defecto a los parámetros, que se usarán si no se proporciona un argumento o si es undefined.

// Parámetros con valores por defecto
function saludar(nombre = "Invitado", saludo = "Hola") {
  console.log(`${saludo}, ${nombre}!`);
}

saludar();  // Muestra: Hola, Invitado!
saludar("María");  // Muestra: Hola, María!
saludar("Juan", "Bienvenido");  // Muestra: Bienvenido, Juan!

// Útil para configuraciones opcionales
function crearElemento(tipo = "div", texto = "", clases = []) {
  console.log(`Creando ${tipo} con texto "${texto}" y clases ${clases}`);
}

crearElemento();  // Usa todos los valores por defecto
crearElemento("p", "Hola mundo");  // Usa el valor por defecto para clases

Los parámetros por defecto son una característica de ES6 (ECMAScript 2015) que hace que tu código sea más robusto y fácil de usar.

Parámetros Rest

Los parámetros rest te permiten representar un número indefinido de argumentos como un array, usando la sintaxis ...nombreParametro.

// Función con parámetro rest
function sumarTodos(...numeros) {
  let total = 0;
  for (let numero of numeros) {
    total += numero;
  }
  return total;
}

console.log(sumarTodos(1, 2));  // Muestra: 3
console.log(sumarTodos(1, 2, 3, 4, 5));  // Muestra: 15

// Combinando parámetros normales y rest
function mostrarEquipo(lider, ...miembros) {
  console.log(`Líder: ${lider}`);
  console.log(`Miembros: ${miembros.join(", ")}`);
}

mostrarEquipo("Ana", "Carlos", "Elena", "María");
// Muestra:
// Líder: Ana
// Miembros: Carlos, Luis, María

El parámetro rest debe ser el último parámetro en la definición de la función, ya que recoge "el resto" de los argumentos.

Desestructuración de Parámetros

Puedes usar la desestructuración para extraer valores de objetos o arrays pasados como parámetros.

// Desestructuración de objetos en parámetros
function mostrarUsuario({ nombre, edad, ciudad = "Desconocida" }) {
  console.log(`${nombre}, ${edad} años, ${ciudad}`);
}

const usuario = {
  nombre: "Pedro",
  edad: 28,
  profesion: "Desarrollador"
};

mostrarUsuario(usuario);  // Muestra: Pedro, 28 años, Desconocida

// Desestructuración con arrays
function procesarCoordenadas([x, y, z = 0]) {
  console.log(`X: ${x}, Y: ${y}, Z: ${z}`);
  return Math.sqrt(x*x + y*y + z*z);
}

const punto2D = [5, 12];
const punto3D = [3, 4, 5];

console.log("Distancia 2D:", procesarCoordenadas(punto2D));
console.log("Distancia 3D:", procesarCoordenadas(punto3D));

La desestructuración hace que tu código sea más limpio y expresivo, especialmente cuando trabajas con objetos complejos.

Ejemplos Prácticos

Veamos algunos ejemplos prácticos de funciones con diferentes tipos de parámetros y valores de retorno:

Calculadora de Descuentos

// Función que calcula el precio final con descuento
function calcularPrecioConDescuento(precio, descuento = 0, impuesto = 0.16) {
  // Validación básica
  if (precio <= 0) {
    return "El precio debe ser mayor que cero";
  }
  
  if (descuento < 0 || descuento > 100) {
    return "El descuento debe estar entre 0 y 100";
  }
  
  // Cálculos
  const precioConDescuento = precio * (1 - descuento / 100);
  const precioFinal = precioConDescuento * (1 + impuesto);
  
  return {
    precioOriginal: precio,
    descuento: descuento,
    precioConDescuento: precioConDescuento,
    impuesto: impuesto,
    precioFinal: precioFinal
  };
}

// Ejemplos de uso
console.log(calcularPrecioConDescuento(100, 20));
// Precio original: 100, con 20% de descuento y 16% de impuesto

console.log(calcularPrecioConDescuento(50));
// Precio original: 50, sin descuento y 16% de impuesto

console.log(calcularPrecioConDescuento(200, 10, 0.21));
// Precio original: 200, con 10% de descuento y 21% de impuesto

Filtrado de Datos

// Función que filtra un array de objetos
function filtrarProductos(productos, { 
  precioMinimo = 0, 
  precioMaximo = Infinity,
  categoria = null,
  ordenarPor = "precio",
  ascendente = true
} = {}) {
  // Primero filtramos
  let resultado = productos.filter(producto => {
    let cumplePrecio = producto.precio >= precioMinimo && 
                       producto.precio <= precioMaximo;
    
    let cumpleCategoria = categoria === null || 
                         producto.categoria === categoria;
    
    return cumplePrecio && cumpleCategoria;
  });
  
  // Luego ordenamos
  resultado.sort((a, b) => {
    let valorA = a[ordenarPor];
    let valorB = b[ordenarPor];
    
    if (ascendente) {
      return valorA - valorB;
    } else {
      return valorB - valorA;
    }
  });
  
  return resultado;
}

// Ejemplo de uso
const productos = [
  { nombre: "Laptop", precio: 1200, categoria: "Electrónica" },
  { nombre: "Teléfono", precio: 800, categoria: "Electrónica" },
  { nombre: "Mesa", precio: 350, categoria: "Muebles" },
  { nombre: "Silla", precio: 150, categoria: "Muebles" }
];

// Filtrar productos de Electrónica entre 500 y 1000
const resultado = filtrarProductos(productos, {
  precioMinimo: 500,
  precioMaximo: 1000,
  categoria: "Electrónica"
});

console.log(resultado);
// [{ nombre: "Teléfono", precio: 800, categoria: "Electrónica" }]

Consejo: Diseña tus funciones para que sean flexibles pero con valores por defecto sensatos. Esto permite que sean fáciles de usar en casos simples, pero también potentes para casos más complejos.

Funciones Anónimas y Funciones Flecha

Además de la declaración tradicional de funciones, JavaScript ofrece formas más concisas de crear funciones: las funciones anónimas y las funciones flecha. Estas son especialmente útiles para funciones cortas o que se pasan como argumentos a otras funciones.

Funciones Anónimas

Las funciones anónimas son funciones sin nombre que a menudo se asignan a variables o se pasan como argumentos a otras funciones.

// Función anónima asignada a una variable
let saludar = function(nombre) {
  return `¡Hola, ${nombre}!`;
};

console.log(saludar("Elena"));  // Muestra: ¡Hola, Elena!

// Función anónima como argumento
setTimeout(function() {
  console.log("Han pasado 2 segundos");
}, 2000);

// Función anónima como callback
[1, 2, 3].map(function(numero) {
  return numero * 2;
});  // Devuelve [2, 4, 6]

// Función anónima autoinvocada (IIFE)
(function() {
  let mensaje = "Esta variable está encapsulada";
  console.log(mensaje);
})();  // Se ejecuta inmediatamente

Las funciones anónimas son útiles cuando necesitas una función temporal o cuando quieres encapsular variables dentro de un ámbito local.

Funciones Flecha

Las funciones flecha (arrow functions) son una sintaxis más corta para escribir funciones, introducida en ES6. Usan el operador => (flecha).

// Sintaxis básica de función flecha
const saludar = (nombre) => {
  return `¡Hola, ${nombre}!`;
};

// Versión más corta para funciones de una sola línea
const saludarCorto = nombre => `¡Hola, ${nombre}!`;

console.log(saludarCorto("Carlos"));  // Muestra: ¡Hola, Carlos!

// Sin parámetros
const decirHola = () => "¡Hola, mundo!";

// Con múltiples parámetros
const sumar = (a, b) => a + b;

// Función flecha como callback
[1, 2, 3].map(numero => numero * 2);  // Devuelve [2, 4, 6]

// Función flecha con cuerpo de bloque
const calcularArea = (radio) => {
  const PI = 3.14159;
  return PI * radio * radio;
};

Las funciones flecha no solo son más cortas, también tienen un comportamiento diferente con respecto a this y no tienen su propio objeto arguments.

Ventajas de las Funciones Flecha

Las funciones flecha ofrecen varias ventajas sobre las funciones tradicionales:

  • Sintaxis más concisa, especialmente para funciones simples
  • No tienen su propio this, usan el this del contexto circundante
  • Ideal para callbacks y funciones que no necesitan su propio this
  • Más legibles en cadenas de métodos o callbacks anidados
// Problema con 'this' en funciones tradicionales
const persona = {
  nombre: "Ana",
  amigos: ["Carlos", "Elena", "Luis"],
  saludarAmigos: function() {
    // 'this' aquí se refiere a 'persona'
    this.amigos.forEach(function(amigo) {
      // 'this' aquí NO se refiere a 'persona'
      console.log(`${this.nombre} saluda a ${amigo}`);  // Error
    });
  }
};

// Solución con función flecha
const personaCorrecta = {
  nombre: "Ana",
  amigos: ["Carlos", "Elena", "Luis"],
  saludarAmigos: function() {
    // 'this' aquí se refiere a 'personaCorrecta'
    this.amigos.forEach(amigo => {
      // 'this' aquí TAMBIÉN se refiere a 'personaCorrecta'
      console.log(`${this.nombre} saluda a ${amigo}`);  // Funciona
    });
  }
};

Limitaciones de las Funciones Flecha

A pesar de sus ventajas, las funciones flecha no son adecuadas para todos los casos:

  • No pueden usarse como constructores (con new)
  • No tienen su propio objeto arguments
  • No pueden usarse como métodos de objetos si necesitas acceder a this del objeto
  • No tienen la propiedad prototype
  • No pueden usarse como generadores (con function*)
// No se puede usar como constructor
const Persona = (nombre) => {
  this.nombre = nombre;  // No funciona como esperarías
};

// Esto causará un error
// const ana = new Persona("Ana");  // TypeError

// No tiene su propio 'arguments'
const sumarTodos = () => {
  // Esto causará un error
  // return Array.from(arguments).reduce((a, b) => a + b, 0);
};

// Solución: usar parámetros rest
const sumarTodosCorrecta = (...args) => {
  return args.reduce((a, b) => a + b, 0);
};

Consejo: Usa funciones flecha para callbacks y funciones cortas. Para métodos de objetos y funciones que necesitan su propio this, usa la sintaxis de función tradicional.

Manipulación de Arrays

// Datos de ejemplo
const estudiantes = [
  { nombre: "Ana", edad: 22, calificaciones: [85, 90, 78] },
  { nombre: "Carlos", edad: 19, calificaciones: [92, 88, 95] },
  { nombre: "Elena", edad: 20, calificaciones: [75, 80, 85] },
  { nombre: "David", edad: 21, calificaciones: [88, 84, 90] }
];

// Filtrar estudiantes mayores de 20 años
const mayoresDe20 = estudiantes.filter(est => est.edad > 20);

// Obtener solo los nombres
const nombres = estudiantes.map(est => est.nombre);

// Calcular promedio de calificaciones para cada estudiante
const promedios = estudiantes.map(est => {
  const suma = est.calificaciones.reduce((a, b) => a + b, 0);
  const promedio = suma / est.calificaciones.length;
  return {
    nombre: est.nombre,
    promedio: promedio.toFixed(1)
  };
});

// Encontrar al estudiante con el promedio más alto
const mejorEstudiante = promedios.reduce((mejor, actual) => {
  return parseFloat(actual.promedio) > parseFloat(mejor.promedio) 
    ? actual 
    : mejor;
}, promedios[0]);

console.log(mejorEstudiante);  // { nombre: "Carlos", promedio: "91.7" }

Manejo de Eventos

// Ejemplo de manejo de eventos con funciones flecha
document.addEventListener('DOMContentLoaded', () => {
  // Seleccionar elementos
  const boton = document.getElementById('miBoton');
  const input = document.getElementById('miInput');
  const lista = document.getElementById('miLista');
  
  // Agregar evento al botón
  boton.addEventListener('click', () => {
    const texto = input.value.trim();
    
    if (texto) {
      // Crear nuevo elemento de lista
      const nuevoItem = document.createElement('li');
      nuevoItem.textContent = texto;
      
      // Agregar botón para eliminar
      const btnEliminar = document.createElement('button');
      btnEliminar.textContent = 'Eliminar';
      btnEliminar.addEventListener('click', () => {
        nuevoItem.remove();
      });
      
      nuevoItem.appendChild(btnEliminar);
      lista.appendChild(nuevoItem);
      
      // Limpiar input
      input.value = '';
      input.focus();
    }
  });
  
  // Permitir agregar con Enter
  input.addEventListener('keypress', evento => {
    if (evento.key === 'Enter') {
      boton.click();
    }
  });
});

Consejo: Las funciones flecha brillan especialmente en operaciones de arrays como map, filter, reduce y en callbacks para eventos, donde su sintaxis concisa hace que el código sea más legible.

Ejercicio 1: Conversor de Temperatura

Escribe una función que convierta grados Celsius a Fahrenheit y otra que convierta Fahrenheit a Celsius. Luego crea una función principal que use ambas según un parámetro de tipo.

Solución:
// Conversor de temperatura
// Celsius a Fahrenheit: (C × 9/5) + 32
const celsiusAFahrenheit = celsius => (celsius * 9/5) + 32;

// Fahrenheit a Celsius: (F - 32) × 5/9
const fahrenheitACelsius = fahrenheit => (fahrenheit - 32) * 5/9;

// Función principal
function convertirTemperatura(valor, tipo) {
  if (tipo === 'C') {
    return {
      original: `${valor}°C`,
      convertido: `${celsiusAFahrenheit(valor).toFixed(1)}°F`
    };
  } else if (tipo === 'F') {
    return {
      original: `${valor}°F`,
      convertido: `${fahrenheitACelsius(valor).toFixed(1)}°C`
    };
  } else {
    return "Tipo de temperatura no válido. Use 'C' o 'F'.";
  }
}

// Ejemplos de uso
console.log(convertirTemperatura(25, 'C'));  // 25°C a 77.0°F
console.log(convertirTemperatura(68, 'F'));  // 68°F a 20.0°C

Ejercicio 2: Analizador de Texto

Crea una función que analice un texto y devuelva estadísticas como el número de palabras, caracteres, y la palabra más frecuente. Usa funciones flecha para las operaciones internas.

Solución:
// Analizador de texto
function analizarTexto(texto) {
  // Si no hay texto, devolver objeto vacío
  if (!texto || texto.trim() === '') {
    return {
      caracteres: 0,
      palabras: 0,
      frases: 0,
      palabraMasComun: 'N/A',
      frecuenciaPalabraMasComun: 0
    };
  }
  
  // Contar caracteres (sin espacios)
  const caracteres = texto.replace(/\s/g, '').length;
  
  // Contar palabras
  const palabras = texto.trim().split(/\s+/).length;
  
  // Contar frases (terminadas en punto, exclamación o interrogación)
  const frases = texto.split(/[.!?]+/).filter(frase => frase.trim() !== '').length;
  
  // Encontrar palabra más común
  const contadorPalabras = {};
  const palabrasArray = texto.toLowerCase().match(/\b\w+\b/g) || [];
  
  palabrasArray.forEach(palabra => {
    contadorPalabras[palabra] = (contadorPalabras[palabra] || 0) + 1;
  });
  
  let palabraMasComun = '';
  let frecuenciaPalabraMasComun = 0;
  
  Object.entries(contadorPalabras).forEach(([palabra, frecuencia]) => {
    if (frecuencia > frecuenciaPalabraMasComun) {
      palabraMasComun = palabra;
      frecuenciaPalabraMasComun = frecuencia;
    }
  });
  
  return {
    caracteres,
    palabras,
    frases,
    palabraMasComun,
    frecuenciaPalabraMasComun
  };
}

// Ejemplo de uso
const texto = "JavaScript es un lenguaje de programación muy versátil. JavaScript se utiliza tanto en el frontend como en el backend. Las funciones en JavaScript son muy poderosas.";
console.log(analizarTexto(texto));
