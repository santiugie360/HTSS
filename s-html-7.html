Introducción a los Formularios

Los formularios son elementos fundamentales en la web que permiten a los usuarios interactuar con las páginas y enviar información. Desde inicios de sesión hasta encuestas, los formularios son la principal herramienta para la recopilación de datos.

Recopilación de Datos

Los formularios permiten recopilar información de los usuarios de manera estructurada y organizada para su posterior procesamiento.

Interactividad

Proporcionan una forma directa de interacción entre el usuario y la aplicación web, mejorando la experiencia del usuario.

Validación

Permiten implementar validaciones para asegurar que los datos ingresados cumplen con los requisitos necesarios antes de ser enviados.

La Etiqueta <form>

Estructura básica

<form action="/procesar-formulario" method="post">
  <!-- Elementos del formulario -->
  <input type="text" name="nombre">
  <input type="submit" value="Enviar">
</form>

La etiqueta <form> es el contenedor principal para todos los elementos de un formulario. Define un formulario que se utiliza para recopilar datos de entrada del usuario.

Características principales:

  • Actúa como contenedor para todos los controles de formulario
  • Define cómo y dónde se enviarán los datos
  • Permite agrupar elementos relacionados
  • Facilita la validación de datos
  • Mejora la accesibilidad y la experiencia del usuario

Ejemplo visual de un formulario básico

Atributos principales

1 action

Especifica la URL a la que se enviarán los datos del formulario cuando se envíe.

<form action="/procesar-datos.php">...</form>

Si se omite, los datos se envían a la página actual.

2 method

Define el método HTTP para enviar los datos del formulario.

<form method="post">...</form>

Valores comunes: "get" (por defecto) y "post".

3 name

Especifica un nombre para el formulario, útil para referenciarlo desde scripts.

<form name="registro">...</form>

Atributos adicionales

4 enctype

Especifica cómo se deben codificar los datos del formulario al enviarlos.

<form enctype="multipart/form-data">...</form>

Necesario cuando se incluyen archivos en el formulario.

5 target

Especifica dónde mostrar la respuesta recibida después de enviar el formulario.

<form target="_blank">...</form>

Valores: _blank, _self, _parent, _top, o nombre de un iframe.

6 autocomplete

Indica si el formulario debe tener la función de autocompletado activada.

<form autocomplete="off">...</form>

Valores: "on" (por defecto) u "off".

Métodos HTTP: GET vs POST

GET

  • Los datos se envían como parte de la URL
  • Visible en la barra de direcciones del navegador
  • Limitado a aproximadamente 2000 caracteres
  • No seguro para datos sensibles
  • Se puede marcar como favorito
  • Ideal para búsquedas y filtros

Ejemplo de URL con método GET:

https://ejemplo.com/buscar?q=html&categoria=web

POST

  • Los datos se envían en el cuerpo de la solicitud HTTP
  • No visible en la URL
  • Sin limitación de tamaño
  • Más seguro para datos sensibles
  • No se puede marcar como favorito
  • Ideal para envío de formularios con datos personales

Ejemplo de código con método POST:

<form method="post" action="/procesar">...</form>

¿Cuándo usar cada método?

  • GET: Para operaciones de solo lectura, como búsquedas o filtros.
  • POST: Para operaciones que modifican datos, envían información sensible o archivos.

Campos de Entrada

La Etiqueta <input>

<input type="text" name="nombre" placeholder="Escribe tu nombre">

La etiqueta <input> es el elemento más utilizado en formularios. Crea diferentes tipos de controles de entrada dependiendo del atributo type.

Atributos comunes:

  • type: Define el tipo de campo (text, password, email, etc.)
  • name: Identifica el campo cuando se envía el formulario
  • value: Especifica el valor inicial del campo
  • placeholder: Muestra un texto de ejemplo en el campo
  • required: Indica que el campo es obligatorio
  • disabled: Desactiva el campo
  • readonly: Hace que el campo sea de solo lectura

Ejemplos:

Tipos de Input: Texto

1 text

Campo básico para entrada de texto de una sola línea.

<input type="text" name="nombre" placeholder="Tu nombre">

2 password

Campo para contraseñas, oculta los caracteres ingresados.

<input type="password" name="clave" placeholder="Contraseña">

3 email

Campo para direcciones de correo electrónico con validación básica.

<input type="email" name="correo" placeholder="tu@email.com">

Tipos de Input: Numéricos

4 number

Campo para valores numéricos con controles para incrementar/decrementar.

<input type="number" name="edad" min="18" max="100">

5 range

Control deslizante para seleccionar un valor dentro de un rango.

<input type="range" name="volumen" min="0" max="100" step="10">

6 date

Campo para seleccionar una fecha con un selector de calendario.

<input type="date" name="fecha_nacimiento">

Tipos de Input: Selección

7 checkbox

Casilla de verificación que permite seleccionar múltiples opciones.

<input type="checkbox" name="acepto" id="acepto">
<label for="acepto">Acepto los términos</label>

8 radio

Botón de opción que permite seleccionar una sola opción de un grupo.

<input type="radio" name="genero" id="masculino" value="M">
<label for="masculino">Masculino</label>
<input type="radio" name="genero" id="femenino" value="F">
<label for="femenino">Femenino</label>

9 color

Selector de color que muestra una paleta de colores.

<input type="color" name="color_favorito" value="#ff6b00">

Tipos de Input: Especiales

10 file

Campo para seleccionar y subir archivos.

<input type="file" name="documento" accept=".pdf,.doc">

11 hidden

Campo oculto que no se muestra pero envía datos con el formulario.

<input type="hidden" name="id_usuario" value="12345">

Este campo no es visible para el usuario pero envía el valor al servidor.

12 submit y reset

Botones para enviar o reiniciar el formulario.

<input type="submit" value="Enviar">
<input type="reset" value="Limpiar">

Atributos de Validación

HTML5 introdujo varios atributos que permiten validar los datos del formulario directamente en el navegador, sin necesidad de JavaScript.

required

Especifica que un campo debe ser completado antes de enviar el formulario.

<input type="text" name="usuario" required>

pattern

Define un patrón (expresión regular) que debe cumplir el valor del campo.

<input type="text" name="codigo" pattern="[A-Z]{3}[0-9]{4}" 
       placeholder="Formato: ABC1234">

Elementos de Formulario

Etiqueta <label>

Define una etiqueta para un elemento de formulario. Mejora la usabilidad y accesibilidad.

Uso básico

<label for="nombre">Nombre:</label>
<input type="text" id="nombre" name="nombre">

Etiqueta <textarea>

Crea un campo de texto multilínea para ingresar texto largo.

Sintaxis básica

<textarea name="comentario" rows="4" cols="50">
Texto inicial (opcional)
</textarea>

Etiqueta <select>

Crea una lista desplegable que permite seleccionar una o varias opciones.

Lista desplegable simple

<select name="pais">
  <option value="es">España</option>
  <option value="mx">México</option>
  <option value="ar">Argentina</option>
  <option value="co">Colombia</option>
</select>

Etiqueta <button>

Crea un botón que puede contener texto y elementos HTML, a diferencia de <input type="button">.

Tipos de botones

<button type="submit">Enviar formulario</button>
<button type="reset">Reiniciar formulario</button>
<button type="button">Botón genérico</button>
