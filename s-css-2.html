Selectores y Especificidad en CSS

Aprende a seleccionar elementos HTML con precisión y entender cómo se aplican las reglas CSS

Introducción a los Selectores CSS

Los selectores son patrones que se utilizan para seleccionar y aplicar estilos a elementos HTML específicos. Son la base fundamental de CSS, ya que nos permiten dirigir nuestros estilos exactamente donde queremos aplicarlos.

Precisión

Los selectores permiten apuntar a elementos específicos en la página, desde elementos individuales hasta grupos con características comunes.

Jerarquía

La especificidad determina qué reglas CSS tienen prioridad cuando varias reglas apuntan al mismo elemento.

Reutilización

Los selectores permiten aplicar los mismos estilos a múltiples elementos, facilitando la consistencia y el mantenimiento del código.

Selectores Básicos

Selector Universal

* {
  margin: 0;
  padding: 0;
  box-sizing: border-box;
}

El selector universal * selecciona todos los elementos HTML en la página. Es útil para restablecer estilos predeterminados o aplicar propiedades globales.

Características:

  • Selecciona todos los elementos HTML
  • Tiene la especificidad más baja (0,0,0)
  • Puede afectar al rendimiento si se usa con muchas propiedades
  • Comúnmente usado en "reset CSS"

Selector de Tipo

p {
  color: #663399;
  line-height: 1.6;
}

h1 {
  font-size: 2rem;
  margin-bottom: 1rem;
}

El selector de tipo selecciona todos los elementos HTML que coinciden con el nombre del elemento especificado.

Características:

  • Selecciona por nombre de etiqueta HTML
  • Afecta a todos los elementos de ese tipo
  • Especificidad baja (0,0,1)
  • Ejemplos: p, div, h1, span

Selector de Clase

.destacado {
  background-color: #663399;
  color: white;
  padding: 10px;
}

.grande {
  font-size: 1.5rem;
}

El selector de clase selecciona todos los elementos que tienen el atributo class con el valor especificado.

Características:

  • Se identifica con un punto . seguido del nombre de la clase
  • Un elemento puede tener múltiples clases
  • Especificidad media (0,1,0)
  • Muy versátil y ampliamente utilizado

Selector de ID

#principal {
  border: 2px solid #663399;
  max-width: 800px;
  margin: 0 auto;
}

#boton-enviar {
  background-color: #663399;
  color: white;
}

El selector de ID selecciona un elemento basado en el valor de su atributo id. Los IDs deben ser únicos en la página.

Características:

  • Se identifica con un hash # seguido del nombre del ID
  • Cada ID debe ser único en la página
  • Especificidad alta (1,0,0)
  • Útil para elementos únicos como encabezados o secciones principales

Selector de Atributo

[type="text"] {
  border: 1px solid #ccc;
  padding: 5px;
}

[href^="https"] {
  color: #663399;
}

El selector de atributo selecciona elementos basados en la presencia o valor de un atributo.

Variantes:

  • [attr] - Elementos con el atributo attr
  • [attr="valor"] - Elementos donde attr es exactamente "valor"
  • [attr^="valor"] - Elementos donde attr comienza con "valor"
  • [attr$="valor"] - Elementos donde attr termina con "valor"
  • [attr*="valor"] - Elementos donde attr contiene "valor"

Agrupación de Selectores

h1, h2, h3 {
  font-family: 'Arial', sans-serif;
  color: #663399;
}

.alerta, .error, .exito {
  padding: 10px;
  border-radius: 4px;
  margin-bottom: 15px;
}

La agrupación de selectores permite aplicar el mismo conjunto de reglas a múltiples selectores, separándolos con comas.

Ventajas:

  • Reduce la repetición de código
  • Facilita el mantenimiento
  • Mejora la legibilidad
  • Reduce el tamaño del archivo CSS

Selectores Avanzados

Selector Descendiente

article p {
  font-size: 16px;
  line-height: 1.6;
}

.menu li {
  list-style: none;
  padding: 5px 0;
}

El selector descendiente selecciona elementos que son descendientes (hijos, nietos, etc.) de otro elemento.

Características:

  • Se escribe con un espacio entre selectores
  • Selecciona elementos anidados a cualquier nivel
  • Útil para estilos contextuales
  • Ejemplo: div p selecciona todos los párrafos dentro de divs

Selector de Hijo Directo

ul > li {
  border-bottom: 1px solid #ddd;
}

.contenedor > p {
  font-weight: bold;
}

El selector de hijo directo selecciona elementos que son hijos directos de otro elemento, sin incluir otros descendientes.

Características:

  • Se escribe con el símbolo > entre selectores
  • Solo selecciona hijos directos (primer nivel)
  • Más específico que el selector descendiente
  • Ejemplo: ul > li selecciona solo los li que son hijos directos de ul
  • Elemento de lista - hijo directo
  • Otro elemento de lista - hijo directo
    • Elemento de lista anidado (no es hijo directo)

Selector de Hermano Adyacente

h2 + p {
  font-size: 18px;
  font-style: italic;
}

label + input {
  margin-left: 10px;
}

El selector de hermano adyacente selecciona un elemento que está inmediatamente después de otro elemento, cuando ambos comparten el mismo padre.

Características:

  • Se escribe con el símbolo + entre selectores
  • Solo selecciona el elemento inmediatamente siguiente
  • Ambos elementos deben compartir el mismo padre
  • Ejemplo: h2 + p selecciona párrafos que vienen justo después de un h2

Selector de Hermanos Generales

h2 ~ p {
  color: #663399;
}

.activo ~ li {
  opacity: 0.7;
}

El selector de hermanos generales selecciona todos los elementos que son hermanos posteriores de otro elemento, cuando comparten el mismo padre.

Características:

  • Se escribe con el símbolo ~ entre selectores
  • Selecciona todos los elementos hermanos que siguen
  • No tienen que ser inmediatamente adyacentes
  • Ejemplo: h2 ~ p selecciona todos los párrafos que vienen después de un h2

Pseudo-clases

Las pseudo-clases son selectores que permiten seleccionar elementos basados en estados o posiciones que no pueden ser expresados en el marcado HTML.

Pseudo-clases comunes:

Estados de interacción
  • :hover - Cuando el cursor está sobre el elemento
  • :active - Cuando el elemento está siendo activado
  • :focus - Cuando el elemento tiene el foco
  • :visited - Enlaces que ya han sido visitados
Estados de formulario
  • :checked - Elementos de formulario marcados
  • :disabled - Elementos de formulario deshabilitados
  • :valid - Elementos con valor válido
  • :invalid - Elementos con valor inválido

Pseudo-clases estructurales:

Posición
  • :first-child - Primer hijo de su padre
  • :last-child - Último hijo de su padre
  • :nth-child(n) - Enésimo hijo de su padre
  • :nth-of-type(n) - Enésimo elemento de su tipo
Ejemplos de código
li:nth-child(odd) {
  background-color: #f2f2f2;
}

p:first-child {
  font-weight: bold;
}

input:focus {
  border-color: #663399;
  outline: none;
}

Pseudo-elementos

Los pseudo-elementos permiten seleccionar y estilizar partes de un elemento que no existen explícitamente en el HTML.

Pseudo-elementos comunes:

  • ::before - Inserta contenido antes del contenido del elemento
  • ::after - Inserta contenido después del contenido del elemento
  • ::first-letter - Selecciona la primera letra del texto
  • ::first-line - Selecciona la primera línea del texto
  • ::selection - Selecciona la porción de texto seleccionada por el usuario
  • ::placeholder - Selecciona el texto de marcador de posición en inputs
Ejemplos de código
.cita::before {
  content: """;
  font-size: 2em;
  color: #663399;
}

.cita::after {
  content: """;
  font-size: 2em;
  color: #663399;
}

p::first-letter {
  font-size: 1.5em;
  font-weight: bold;
  color: #663399;
}

Especificidad y Cascada

¿Qué es la Especificidad?

La especificidad es el mecanismo que utiliza el navegador para determinar qué reglas CSS tienen prioridad cuando varias reglas apuntan al mismo elemento y propiedad. Es un cálculo que determina qué estilos se aplican finalmente.

Importancia de la especificidad:

  • Determina qué estilos prevalecen en caso de conflicto
  • Ayuda a organizar y estructurar el CSS
  • Evita el uso excesivo de !important
  • Facilita el mantenimiento y la depuración

Ejemplo de conflicto:

/* Especificidad: 0,0,1 */
p {
  color: blue;
}

/* Especificidad: 0,1,0 */
.texto {
  color: red;
}

/* ¿Qué color se aplicará? */

Este texto será rojo porque la clase tiene mayor especificidad que el selector de tipo.

Cálculo de Especificidad

La especificidad se calcula como un valor de cuatro partes, generalmente representado como cuatro números separados por comas: (a,b,c,d)

Componentes de la especificidad:

a - Estilos en línea

Valor: 1,0,0,0

<p style="color: red;">Texto</p>
b - IDs

Valor: 0,1,0,0 por cada ID

#header { color: blue; }
c - Clases, atributos y pseudo-clases

Valor: 0,0,1,0 por cada uno

.active { color: green; }
[type="text"] { color: gray; }
:hover { color: purple; }
d - Elementos y pseudo-elementos

Valor: 0,0,0,1 por cada uno

p { color: black; }
::before { content: "→"; }

Reglas de Cascada

La cascada es el algoritmo que determina qué estilos se aplican cuando hay múltiples reglas para un elemento. Sigue un orden específico de prioridad.

Orden de prioridad (de mayor a menor):

  1. Importancia
    • Declaraciones con !important
    • Estilos normales
  2. Origen
    • Estilos del autor (tu CSS)
    • Estilos del usuario
    • Estilos del navegador (user agent)
  3. Especificidad
    • Calculada según las reglas anteriores
  4. Orden de aparición
    • La última regla declarada gana

Ejemplo de cascada:

/* Regla 1 */
p {
  color: blue;
}

/* Regla 2 */
p {
  color: red;
}

/* La regla 2 gana por orden de aparición */

!important y Buenas Prácticas

La declaración !important

p {
  color: red !important; /* Esto anulará otras reglas */
}

#contenido p {
  color: blue; /* Esta regla será ignorada */
}

La declaración !important anula todas las demás reglas, independientemente de su especificidad, excepto otras reglas con !important.

Buenas prácticas:

  • Evita usar !important siempre que sea posible
  • Utiliza selectores más específicos en lugar de !important
  • Organiza tu CSS para minimizar conflictos
  • Usa metodologías como BEM para evitar problemas de especificidad
  • Comprende la especificidad para escribir CSS más predecible
  • Utiliza herramientas de análisis para detectar problemas de especificidad

Herencia en CSS

La herencia es el mecanismo por el cual algunas propiedades CSS se transmiten de elementos padres a elementos hijos. No todas las propiedades se heredan.

Propiedades que se heredan:

  • color
  • font-family
  • font-size
  • font-weight
  • line-height
  • text-align
  • letter-spacing
  • visibility

Propiedades que NO se heredan:

  • margin
  • padding
  • border
  • background
  • height, width
  • position
  • display
  • z-index

Valores especiales para herencia:

inherit

Fuerza la herencia de una propiedad que normalmente no se hereda.

button {
  border: inherit; /* Hereda el borde del padre */
}
initial

Establece el valor inicial definido por la especificación CSS.

p {
  color: initial; /* Vuelve al color por defecto */
}
unset

Actúa como inherit para propiedades heredadas y como initial para las no heredadas.

div {
  all: unset; /* Resetea todas las propiedades */
}

¿Qué significa esto?

La especificidad determina qué reglas CSS tienen prioridad. Un selector con mayor especificidad anulará a uno con menor especificidad, independientemente del orden en que aparezcan en el CSS.

Quiz: Especificidad CSS

Pregunta 1: ¿Qué selector tiene mayor especificidad?

p.destacado (0,0,1,1)
#contenido p (0,1,0,1)
body .articulo p:first-child (0,0,2,2)

Correcto! El selector #contenido p tiene mayor especificidad (0,1,0,1) porque incluye un ID, que tiene más peso que las clases y pseudo-clases.

Pregunta 2: Ordena estos selectores de menor a mayor especificidad

* (0,0,0,0)
li (0,0,0,1)
ul li (0,0,0,2)
li.active (0,0,1,1)
#nav li.active (0,1,1,1)

¡Correcto! El orden correcto de menor a mayor especificidad es:

  1. * (0,0,0,0)
  2. li (0,0,0,1)
  3. ul li (0,0,0,2)
  4. li.active (0,0,1,1)
  5. #nav li.active (0,1,1,1)
